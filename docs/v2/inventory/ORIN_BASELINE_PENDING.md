# V2 Planning Material: Orin Baseline Pending

Not current runtime truth. This inventory is intentionally pending.

## Why deferred

- Current V2 planning evidence is strongest for Mini and Studio.
- The current phase is command-center synthesis and preserve-vs-rebuild mapping, not host expansion.
- Orin work is explicitly out of scope for the current phase-one planning boundary.

## Minimum future inventory questions

- What OS, kernel, hardware class, RAM, and storage posture does Orin currently have?
- Which ports and services are actively listening, and which of them are part of the current homelab contract versus experiments?
- Is Orin currently carrying any ASR, TTS, or voice-gateway-adjacent runtime that V2 would need to preserve?
- Which model directories, caches, and registries exist on Orin, and how large are they?
- Which service managers, env files, or deployment paths are in use?
- Which surfaces are clearly preserve candidates, stop-later candidates, or unknown/risky?

## Phase required

- This inventory is required before any Orin integration planning or any speech/edge-runtime V2 slice.
